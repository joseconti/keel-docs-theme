3 direcciones de template-topic

La plantilla de contenido es el 95 % de las páginas. Aquí van tres familias estéticas distintas, cada una neutra de marca (su carácter no depende del acento, que cada proyecto cambiará) y con una apuesta arriesgada. Mismo contenido demo — Basalt, plataforma de sincronización de inventario — para comparar en igualdad. Cada una en modo claro y oscuro automático.

Elige una por su id (1a / 1b / 1c). Consolidaré el sistema completo sobre ella conservando sus elementos firma. — Tipos definitivos serán woff2 OFL autoalojados; aquí se previsualizan vía fuente web.

1A

Errata

Manual de ingeniería editorial. Columna de lectura serif, cabecera reglada, numeración de sección en cursiva, índice lateral con filete. Apuesta: tipografía serif (Source Serif 4) para lectura prolongada — poco común en docs de software, aporta autoridad de manual impreso. Tipos: Source Serif 4 · IBM Plex Sans (UI) · IBM Plex Mono (código).

Claro
Oscuro
1B

Console

Instrumento monospace. Interfaz y titulares en monospace (man-page / RFC bien compuesto), rejilla de 1 px, filetes discontinuos, densidad tabular. Apuesta: monospace como tipografía de interfaz y de encabezado, con cuerpo en sans (Public Sans) para no sacrificar la lectura. Tipos: JetBrains Mono (UI+titulares+código) · Public Sans (cuerpo).

Claro
Oscuro
1C

Margins

Libro de texto con marginalia. Grotesca contemporánea, escala fuerte; el índice "en esta página" y los números de sección viven en el margen exterior como notas al margen (Tufte). Apuesta: invierte la relación habitual sidebar/contenido/TOC llevando el índice al margen del propio texto. Tipos: Hanken Grotesk · Spline Sans Mono (código).
